# Starbase Defender

迫りくる敵艦隊から中央の宇宙基地を守り抜くターン制戦術シミュレーション(ローグライク)ゲームです。

プレイヤーは防衛艦を操作し、中央の宇宙基地を破壊しようとする敵を迎撃し、撤退させるのが目的です。
撃墜しないのがポイントです。
移動・攻撃・シールドにはエネルギーが必要で、限られた資源を管理しながら戦略的に立ち回る必要があります。

[**[ Play ]**](https://ytani01.github.io/star-base-defender/)

## 重要事項

* このゲームの目的は、敵機を**撤退**させることです。
  * 全ての敵を撤退させると、襲撃がおさまり、そのステージはクリアされます。
  * もたもたしていると、増援が出現します。

* 敵機を撃墜すると、敵意を煽ってしまい、増援が出現します。つまり、撃墜は逆効果です。

* ハイスコアは、ステージクリアごとに記録されます。ゲームオーバー時(ステージクリアできなかったとき)は、記録はされません。


## 操作方法

ゲームはすべて**マウスクリック（または画面タップ）**で操作します。プレイヤーが1回行動すると、敵のターンに移行します。

|クリック| 動作・結果 |
|-|-|
|空間|クリックした座標へ向かって移動する。障害物がある場合は手前で停止する。|
|基地|ドッキング圏内に移動するとドッキングされ、自機のエネルギーとシールドが最大値まで回復する。ドッキング中は攻撃できない。敵からの攻撃は全て基地が受ける。|
|敵|フェーザー砲発射。射程外や障害物がある場合は、無駄撃ちになる。エネルギーを消費する。|
|自機|シールド回復。エネルギーを消費する。|


## ターンシステム
* プレイヤーが行動（移動、攻撃、ドッキング、または射程外への空撃ちなど）を完了すると、敵のターンになります。
* 全ての敵艦が同時に移動または攻撃を行った後、次のターンに進み、プレイヤーの手番に戻ります。


## 敵の行動AI
* 耐久力が50%以上ある敵は、射程内に基地またはプレイヤーがいれば攻撃（ピンク色のビーム）を仕掛けてきます。射程外の場合は、基地やプレイヤーに向けて接近してきます。

## 勝利・敗北条件

### 勝利条件
* 宇宙域に侵入したすべての敵を撤退させる。
* 達成するとミッション完了となり、次のミッションに進むことができる。

### 敗北条件
* **宇宙基地の陥落**: 宇宙基地のシールドが 0% になる。
* **自機の撃破**: 自機のシールドが 0% になる。
* **自機のエネルギー消失** 自機のエネルギーが 0%  になる。(移動も攻撃も何もできなくなる)

## スコアの記録
* ミッションを完了(ステージクリア)すると、そのたびにスコアが記録される。
* ミッションを完了できずゲームオーバーになった時点のスコアは評価されない。
